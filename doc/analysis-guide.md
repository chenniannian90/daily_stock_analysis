# 自选股智能分析系统 — 功能原理与使用指南

## 一、系统概述

本系统是一个基于多智能体（Multi-Agent）协作的 A股/港股/美股 智能分析平台。每天定时（或手动触发）对自选股列表中的全部股票逐只执行深度分析，生成包含技术面、消息面、风险面的综合决策仪表盘（Decision Dashboard），并通过飞书/微信/Telegram 等渠道推送分析报告。

---

## 二、分析核心原理

### 2.1 七大交易理念

系统的分析逻辑围绕以下七条核心理念构建：

| 编号 | 理念 | 说明 |
|------|------|------|
| 1 | **严进策略** | 不追高，乖离率（现价偏离 MA5 的比例）超过阈值不买 |
| 2 | **趋势交易** | MA5 > MA10 > MA20 多头排列，顺势而为 |
| 3 | **效率优先** | 关注筹码结构好的股票，量能确认趋势有效性 |
| 4 | **买点偏好** | 优先选择回踩 MA5/MA10 均线支撑的时机介入 |
| 5 | **风险排查** | 利空新闻一票否决，重大风险直接降级买入信号 |
| 6 | **量价配合** | 成交量必须验证价格运动方向 |
| 7 | **强势趋势补偿** | 龙头股在强势多头排列时可适当放宽乖离率标准 |

### 2.2 综合评分系统（0-100 分）

每日对每只股票计算综合情绪评分，由六个维度加权合成：

| 维度 | 满分 | 核心逻辑 |
|------|------|----------|
| **趋势判断** | 30 分 | 多头排列得分高，空头排列得分低 |
| **乖离率** | 20 分 | 价格贴近或略低于 MA5 得分高，偏离过远严禁追高 |
| **量能分析** | 15 分 | 缩量回调（洗盘特征）最优，放量下跌最差 |
| **均线支撑** | 10 分 | 价格获得 MA5/MA10 支撑各得 5 分 |
| **MACD 指标** | 15 分 | 零轴上金叉最强，死叉最差 |
| **RSI 指标** | 10 分 | 超卖区域得分高（反弹机会），超买区域得分低 |

### 2.3 操作信号分级

```
评分 ≥ 75 且多头排列 → 强烈买入 ⭐
评分 60-74 且非空头   → 买入     ✅
评分 45-59            → 持有     🔵
评分 30-44            → 观望     🟡
评分 < 30 或空头排列  → 卖出/强烈卖出 ❌
```

## 三、多智能体分析流水线

### 3.1 四种分析模式

| 模式 | 流水线 | 适用场景 |
|------|--------|----------|
| **quick** | 技术面 → 决策 | 最快（~2 次 LLM 调用），只看技术指标 |
| **standard**（默认） | 技术面 → 情报 → 决策 | 日常分析，兼顾技术和消息面 |
| **full** | 技术面 → 情报 → 风险 → 决策 | 完整分析，包含风险排查 |
| **specialist** | 技术面 → 情报 → 风险 → 专项策略 → 决策 | 最深度，激活自定义交易策略 |

### 3.2 各智能体职责

#### 技术面智能体（TechnicalAgent）
- 获取实时行情和历史 K 线数据
- 计算均线排列（MA5/MA10/MA20/MA60）
- 运行 MACD、RSI 等指标分析
- 识别 K 线形态和量能特征
- 输出支撑位/压力位/止损位

#### 情报智能体（IntelAgent）
- 搜索股票相关最新新闻
- 分析市场情绪和热点话题
- 识别正面催化剂和风险事件
- 评估信息面的多空倾向

#### 风险智能体（RiskAgent）
- 排查大股东减持/质押风险
- 检查业绩预亏/变脸预警
- 扫描监管处罚/立案调查
- 评估限售股解禁压力
- 检查估值极端情况（PE > 100 或负值）
- 两级风控：**soft**（降级信号）/ **hard**（否决买入）

#### 决策智能体（DecisionAgent）
- 综合前三阶段的所有意见
- 按权重合成最终决策：技术面 40% + 情报 30% + 风险 30%
- 生成决策仪表盘 JSON
- 输出买入价、止损价、止盈价等精确点位

### 3.3 风控覆盖机制

风险智能体有权在最终阶段覆盖决策：

- **否决买入**：存在高风险标记时，买入信号强制降为持有
- **降级一级**：中等风险时，买入→持有，持有→卖出
- **降级二级**：多重风险叠加时，买入→卖出

被风控覆盖的结果会在分析摘要中标记 `[风控下调: buy -> hold]`。

## 四、关键指标详解

### 4.1 均线与趋势

| 术语 | 含义 |
|------|------|
| **多头排列** | MA5 > MA10 > MA20，三条均线从左上到右下排列，价格在各均线上方运行 |
| **强势多头** | 多头排列且均线间距在扩大，趋势加速 |
| **空头排列** | MA5 < MA10 < MA20，价格在均线下方运行 |
| **均线缠绕** | 三条均线反复交叉，趋势不明朗（盘整） |
| **金叉** | 短期均线上穿长期均线（看涨信号） |
| **死叉** | 短期均线下穿长期均线（看跌信号） |

### 4.2 乖离率（Bias）

```
乖离率 = (现价 - MA5) / MA5 × 100%
```

- **正值**：价格在 MA5 上方，偏离越大追高风险越大
- **负值**：价格在 MA5 下方（回踩），小幅负值为较好买点
- **严进阈值**：乖离率超过 config 中设置的阈值（默认 5%）时严禁追高
- **强势趋势补偿**：强势多头排列时阈值放宽至 1.5 倍

### 4.3 量能分析

| 量能状态 | 含义 | 评价 |
|----------|------|------|
| **缩量回调** | 价格下跌但成交量萎缩 | ⭐ 最优，通常是主力洗盘 |
| **放量上涨** | 价涨量增 | ✅ 良好，多头力量强劲 |
| **量能正常** | 成交量在 5 日均量附近 | 🟡 中性 |
| **缩量上涨** | 价格上涨但量能不足 | ⚠️ 上攻动能不足 |
| **放量下跌** | 价跌量增 | ❌ 最差，主力出货 |

### 4.4 MACD 指标

```
DIF = EMA(12) - EMA(26)
DEA = EMA(DIF, 9)
MACD 柱 = (DIF - DEA) × 2
```

| MACD 状态 | 含义 |
|-----------|------|
| **零轴上金叉** | ⭐ 最强买入信号，DIF 在零轴上方上穿 DEA |
| **金叉** | ✅ DIF 上穿 DEA，趋势向上 |
| **多头** | DIF > DEA > 0，持续上涨中 |
| **上穿零轴** | ⚡ DIF 从负转正，趋势转强 |
| **死叉** | ❌ DIF 下穿 DEA，趋势向下 |
| **下穿零轴** | ⚠️ DIF 从正转负，趋势转弱 |

### 4.5 RSI 指标（相对强弱指数）

```
RSI = 100 - 100/(1 + RS)，其中 RS = 平均涨幅/平均跌幅
```

- **RSI(12) > 70**：超买，短期回调风险高，不宜追高
- **RSI(12) 60-70**：强势，多头力量充足
- **RSI(12) 40-60**：中性，震荡整理中
- **RSI(12) 30-40**：弱势，关注超跌反弹
- **RSI(12) < 30**：超卖，反弹机会大（但需趋势配合）

## 五、决策仪表盘解读

每份分析报告包含以下关键决策信息：

### 5.1 狙击点位（Sniper Points）

| 点位 | 含义 |
|------|------|
| **ideal_buy（理想买入价）** | 系统建议的最佳介入价格，通常对应 MA5/MA10 支撑位或关键支撑位 |
| **secondary_buy（次选买入价）** | 如果理想价位未能触及，次选的介入价位 |
| **stop_loss（止损价）** | **必须严格执行的退出价格**。当股价跌破此价位时，应立即止损离场，控制亏损 |
| **take_profit（止盈价）** | 目标盈利价位，到达后可考虑分批止盈或移动止损 |

### 5.2 止损价意味着什么

止损价是风控的生命线，其含义包括：

1. **最大可承受亏损**：止损价 = 你的入场成本 - 你愿意承受的最大亏损幅度。当你按照 ideal_buy 买入后，如果股价不幸跌破止损价，说明你的交易逻辑已被市场否定，应果断离场。

2. **技术止损依据**：系统给出的止损价通常基于以下技术位：
   - MA20 均线（中期趋势的最后防线）
   - 近期重要低点/前低支撑
   - 筹码密集区下沿

3. **执行纪律**：止损不是建议，是纪律。许多亏损扩大的根源就是"再等等看"——等来的往往是更深的亏损。系统建议止损设在关键支撑下方 2-3%。

4. **动态止损**：如果股价上涨，可以将止损位上移至成本价（保本止损）或跟随 MA10/MA20 上移（移动止损），锁住利润。

### 5.3 持仓建议

系统区分两种状态给出不同建议：

- **空仓（no_position）**：当前未持有该股时的建议
- **持仓（has_position）**：已持有该股时的操作建议

例如：某股票 signal 为 buy，但你的情况不同：
- 空仓者可考虑在 ideal_buy 附近分批建仓
- 持仓者可在回踩不破关键位时考虑加仓

### 5.4 信号与评分对照

| 评分区间 | decision_type | 操作建议 | 说明 |
|----------|---------------|----------|------|
| 80-100 | buy | 买入 | 多条件满足，高确信度 |
| 60-79 | buy | 买入 | 基本条件满足，有轻微瑕疵 |
| 40-59 | hold | 观望/持有 | 信号混合或存在风险 |
| 20-39 | sell | 减仓/卖出 | 趋势偏弱+风险 |
| 0-19 | sell | 卖出 | 重大风险+空头趋势 |

## 六、如何执行分析

### 6.1 定时自动分析

系统默认在每个交易日收盘后自动触发全量分析，具体时间由 `.env` 中的 `ANALYSIS_SCHEDULE` 配置。

### 6.2 手动触发分析

**方式一：通过 Web 界面**
访问 `http://stock.hao246.cn/`，在首页点击"开始分析"按钮。

**方式二：通过 API**
```bash
# 分析单只股票
curl -X POST http://stock.hao246.cn/api/v1/analysis/analyze \
  -H "Content-Type: application/json" \
  -d '{"stock_code": "000001"}'

# 批量分析全部自选股
curl -X POST http://stock.hao246.cn/api/v1/analysis/analyze-all
```

**方式三：Docker 命令**
```bash
# 在服务器上手动触发一次定时分析
docker exec stock-analyzer python main.py --once
```

### 6.3 查看历史分析

- **Web 页面**：访问 `http://stock.hao246.cn/daily-summary`，选择日期查看当日全部分析结果
- **API 接口**：`GET /api/v1/history/daily-summary?date=2026-05-12`
- **历史列表**：`GET /api/v1/history?start_date=2026-05-12&end_date=2026-05-12`

## 七、自定义交易策略

你可以在 `strategies/` 目录下创建 `.yaml` 文件来定义自己的交易策略，无需编写代码。

策略会被 specialist 模式的专项智能体加载执行，并影响最终的决策仪表盘评分和操作建议。

详见 `strategies/README.md`。

## 八、常见问题

### Q: 为什么同一天同一只股票会有多条分析记录？
A: 系统在每日总结 API 中已做去重处理，只保留当天最新的一条分析。如果手动多次触发分析，数据库会保留全部记录（便于回溯），但查询接口返回的是去重后的结果。

### Q: 评分高但操作建议是"观望"？
A: 可能被风险智能体覆盖了。检查分析摘要中是否有 `[风控下调]` 标记，并查看 risk_warning 字段了解具体风险原因。

### Q: 止损价和 ideal_buy 差距很大怎么办？
A: 这说明当前价格距离理想买点较远。如果止损幅度（ideal_buy → stop_loss）超过你能承受的范围，建议等待更好的入场时机，不要勉强交易。

### Q: 分析结果中的"缩量回调"为什么是好信号？
A: 缩量回调通常意味着卖盘枯竭而非主力出逃。放量下跌才是真正的危险信号。缩量回调 + 均线支撑 = 经典的低吸买点形态。
